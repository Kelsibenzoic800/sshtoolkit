## ⚙️ What sshtoolkit does

sshtoolkit is a browser-based SSH toolkit. It helps you handle common SSH jobs from one place:

- Create SSH key pairs
- Check and edit SSH config files
- Prepare server hardening settings
- Build SSH tunnels
- Manage client setup details
- Work fully on the client side
- Keep your data in your browser session

It is built for quick SSH work without extra desktop tools.

## 🧰 Main uses

Use sshtoolkit when you need to:

- Make a new key for a server
- Set up a safer SSH server config
- Tune a client config for faster access
- Create local or remote tunnels
- Review SSH settings before you copy them to a server
- Keep your SSH notes in one tool

## 🔐 Privacy and local use

sshtoolkit runs in your browser. It does not send your SSH data to a server. Your keys and config work stay on your device while you use the app.

This makes it a good fit for:

- Local SSH prep
- Shared or locked-down work PCs
- Fast checks without installing a large tool set
- Users who want a simple browser workflow

## 🗝️ Generate SSH keys

Use the key tool to create new SSH keys for servers and clients.

Typical steps:

1. Select the key type.
2. Choose a safe key size.
3. Start the key generation step.
4. Save the private key in a secure place.
5. Copy the public key to your server.

Keep the private key private. Use the public key on the server.

## 🛡️ Harden an SSH server

Use the server settings tool to prepare a safer sshd config.

Common options include:

- Disable password login
- Limit root access
- Set allowed users
- Reduce weak cipher use
- Set clear timeout values
- Tighten auth rules

This helps you build a cleaner server config before you apply it.

## 🖥️ Configure your SSH client

Use the client config tool to make login easier.

You can set:

- Host name
- User name
- Port
- Key file path
- Jump host settings
- Alias names for fast access

This is useful when you connect to more than one server.

## 🌐 Create SSH tunnels

Use the tunnel tool when you need to move traffic through SSH.

You can set up:

- Local port forwarding
- Remote port forwarding
- SOCKS proxy style access
- Host and port pairs
- Source and target values

This helps when a service sits behind a private network.
